# Apple / Google / Huawei 文案案例库

## Apple

### Slogan 案例
- 照片：「你的回忆，焕然一新。」
- 备忘录：「随时记录，随处记录。」
- 健康：「你的健康，一目了然。」
- 地图：「探索世界，从这里开始。」
- 钱包：「一碰即付，轻松出行。」

### 功能命名风格
- 简洁名词为主：「照片」、「备忘录」、「提醒事项」
- 功能用动宾结构：「美化照片」、「创建备忘录」
- AI 功能直接描述：「智能搜索」、「实况文本」

### 交互提示风格
- 空状态：「这里还什么都没有。拍张照或截个屏，它就会出现。」（温暖 + 引导行动）
- 错误提示：「无法保存。请检查你的网络连接。」（准确 + 有解决方案）
- 权限申请：「"照片"想访问你的相册。你可以在"设置"中随时更改。」（透明 + 告知控制权）

### 更新日志风格
- 亮点前置，功能 + 价值：「照片 - 现在，你可以使用强大的编辑工具来修饰你的照片」
- 简洁条目式，不说废话

## Google

### Slogan 案例
- Google Photos：「你的照片，更智能了。」
- Google Keep：「捕捉想法，轻松回顾。」
- Google Drive：「随时随地，访问你的文件。」

### 功能命名风格
- 动词驱动：「Enhance（增强）」、「Cleanup（清理）」、「Magic Eraser（魔法橡皮擦）」
- AI 功能用品牌词：「Magic Compose」、「PhotoScan」
- 偏好形容词 + 名词：「Smart Storage」、「Suggested Actions」

### 交互提示风格
- 空状态：「你的照片会出现在这里。开始拍摄或使用相机导入。」（直接 + 行动导向）
- 错误提示：「出了点问题。请重试。」（简洁）
- 权限申请：「允许 Google 照片访问你设备上的照片和视频？」（疑问句 + 明确范围）

### 更新日志风格
- 功能名 + 一句话说明：「Magic Editor - 用 AI 重新构思你的照片」
- 偏重技术能力描述

## Huawei

### Slogan 案例
- 图库：「记录美好，发现精彩。」
- 备忘录：「灵感随时记录，想法不再遗漏。」
- 钱包：「一碰即付，智慧出行。」

### 功能命名风格
- 偏好"智慧"系列：「智慧搜图」、「智慧分类」、「智慧推荐」
- 偏好"一"字系列：「一键美化」、「一指分栏」、「一碰传」
- AI 功能强调能力：「AI 消除」、「AI 扩图」、「AI 云增强」

### 交互提示风格
- 空状态：「暂无内容，点击下方按钮开始创建」（简洁 + 引导）
- 错误提示：「操作失败，请检查网络后重试」（专业 + 有建议）
- 权限申请：「应用需要访问您的照片，以便为您提供更好的服务」（价值导向）

### 更新日志风格
- 分类 + 亮点：「【AI 能力升级】新增 AI 消除功能，轻松去除照片中的路人」
- 偏重功能价值和用户体验提升

## 三大平台对比总结

| 维度 | Apple | Google | Huawei |
|------|-------|--------|--------|
| 核心语气 | 温暖 + 人文关怀 | 技术 + 效率 | 智慧 + 全能 |
| 命名风格 | 极简名词 | 动词/形容词驱动 | "智慧"+功能 / "一"+动作 |
| 空状态 | 场景化 + 温暖 | 直接 + 行动导向 | 简洁 + 引导 |
| 错误提示 | 准确 + 解决建议 | 简洁 | 专业 + 解决建议 |
| 更新日志 | 功能 + 价值 | 功能 + 技术能力 | 分类 + 功能 + 价值 |
| AI 表述 | 不强调 AI 概念 | 强调 AI/品牌词 | 强调 AI/智慧 |

## 对系统应用文案的启示

1. **AI 功能命名**：Apple 偏低调直接，Google 用品牌词包装，Huawei 用「智慧/AI」前缀。根据目标平台选择对应策略。
2. **空状态文案**：Apple 的场景化表达最温暖，Google 最直接，Huawei 最简洁。推荐温暖 + 引导行动的组合。
3. **错误提示**：统一采用"说明问题 + 给出建议"的结构，避免空泛的"操作失败"
4. **更新日志**：采用"分类标签 + 功能名 + 一句话价值描述"的三段式
