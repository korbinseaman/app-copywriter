### Apple 文案风格特征
- **极简短句**：偏好 6-11 字的短句，节奏感强
- **对称结构**：常用四字/五字对仗，如"随时记录，随处记录"
- **口语化表达**："iPhone 又在跟你捉迷藏？"
- **不强调技术参数**：用"拍照、拍视频，出手成片"而非"4800 万像素摄像头"
- **人文关怀**："你的回忆，焕然一新"——把产品功能和用户情感连接
- **功能描述**：场景化，如"有 iPhone 镜像功能，在 Mac 上就能看到 iPhone 屏幕画面，还能直接进行操控，不拿起手机也照样用手机"

### 交互提示风格

- **空状态**：「这里还什么都没有。拍张照或截个屏，它就会出现。」（温暖 + 引导行动）
- **错误提示**：「无法保存。请检查你的网络连接。」（准确 + 有解决方案）
- **权限申请**：「"照片"想访问你的相册。你可以在"设置"中随时更改。」（透明 + 告知控制权）

### 更新日志风格
- 亮点前置，功能 + 价值：「照片 - 现在，你可以使用强大的编辑工具来修饰你的照片」
- 简洁条目式，不说废话

## 三大平台对比总结

| 维度 | Apple | Google | Huawei |
|------|-------|--------|--------|
| 核心语气 | 温暖 + 人文关怀 | 技术 + 效率 | 智慧 + 全能 |
| 命名风格 | 极简名词 | 动词/形容词驱动，品牌词包装 | "智慧"+功能 / "一"+动作 |
| Slogan 长度 | 6-11 字为主 | 9-19 字 | 4-14 字 |
| 空状态 | 场景化 + 温暖 | 直接 + 行动导向 | 简洁 + 引导 |
| 错误提示 | 准确 + 解决建议 | 简洁 | 专业 + 解决建议 |
| 更新日志 | 功能 + 价值 | 功能 + 技术能力 | 分类 + 功能 + 价值 |
| AI 表述 | 不强调 AI 概念 | 强调 AI/品牌词 | 强调 AI/智慧 |
| 跨设备文案 | 场景描述为主 | 功能列表为主 | "一"字系列 + 动作描述 |

## 对系统应用文案的启示

1. **AI 功能命名**：Apple 偏低调直接，Google 用品牌词包装，Huawei 用「智慧/AI」前缀。根据目标平台选择对应策略。
2. **Slogan 长度**：Apple 最短（6-11 字），适合按钮和标签；Google 和 Huawei 可稍长（9-14 字），适合宣传页。
3. **空状态文案**：Apple 的场景化表达最温暖，Google 最直接，Huawei 最简洁。推荐温暖 + 引导行动的组合。
4. **错误提示**：统一采用"说明问题 + 给出建议"的结构，避免空泛的"操作失败"。
5. **更新日志**：采用"分类标签 + 功能名 + 一句话价值描述"的三段式。
6. **跨设备协同文案**：Apple 用场景故事（"在 Mac 上就能看到 iPhone 屏幕画面"），Huawei 用"一"字系列（"一抓一放"），各有特色。
