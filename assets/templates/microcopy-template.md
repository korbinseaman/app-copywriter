# 交互提示文案模板

## 空状态

### 结构
```
[情感安抚 / 场景描述] + [引导行动]
```

### 示例
- 「还没有收藏的照片。浏览相册，点击 ❤️ 收藏你喜欢的瞬间。」
- 「这里空空如也。创建第一条备忘录，开始记录你的灵感吧。」

### 原则
- 不要只说"暂无数据"
- 给出下一步行动建议
- 语气温暖但不啰嗦

## 错误提示

### 结构
```
[说明问题] + [给出建议 / 自动处理方案]
```

### 示例
- 「上传失败。请检查网络连接后重试。」
- 「无法识别该图片。尝试选择一张清晰的照片。」
- 「云空间已满。释放空间或升级存储方案后继续备份。」

### 原则
- 不说"操作失败"这种无意义的废话
- 明确告诉用户发生了什么
- 给出可执行的下一步

## 权限申请

### 结构
```
[说明需要什么权限] + [为什么要] + [用户控制权]
```

### 示例
- 「需要访问你的相册，以便为你智能分类照片。你可以在设置中随时关闭此权限。」
- 「为了提供云备份功能，需要访问你的照片。数据将加密传输，你可随时在设置中管理权限。」

### 原则
- 透明：明确说明用途
- 尊重：告知用户可以拒绝/随时关闭
- 价值：说明权限带来的好处

## 操作反馈（Toast / Snackbar）

### 结构
```
[动作确认] + [可选：后续操作]
```

### 示例
- 「已保存到相册」
- 「已删除，可在回收站恢复」
- 「3 张照片已分享」

### 原则
- 不超过 15 字
- 确认结果，不说过程
- 有后续操作时简要提示
